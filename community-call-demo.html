Demo Title
MCP + Copilot Studio: Let Any Agent Explore Power Platform Solutions via Chat

Demo Idea
Export any Power Platform solution, drop the ZIP into an open-source MCP server, and let a Copilot Studio agent answer questions about what's inside: flows, agents, connectors, tables, system prompts. All through natural-language chat. Live demo: I'll connect a 25-tool MCP server to Copilot Studio via Streamable HTTP, then ask the agent to explore a real solution. The highlight: asking the agent to read another agent's system prompt. AI introspecting AI. Technique-focused, fully open-source (MIT), zero custom connectors needed. The pattern is reusable for any structured data.

Timed Demo Script 12 minutes

0:00 to 1:30
Segment 1: Problem Statement
What to Say
  • "You export a Power Platform solution... now what? You get a ZIP with 30+ subfolders, XML, JSON, YAML files."
  • "Manual inspection is slow. New team members can't understand what's inside."
  • "What if an AI agent could answer any question about your solution, instantly, via chat?"
What to Show
  • Slide: problem statement + screenshot of messy extracted folder
  • Quick scroll through raw XML/JSON to emphasize complexity
1:30 to 3:00
Segment 2: What is MCP?
What to Say
  • "MCP (Model Context Protocol) is an open standard for connecting AI to tools and data."
  • "Think USB-C for AI: any client, any server, one protocol."
  • "Copilot Studio supports MCP natively via Streamable HTTP."
  • "This is NOT a product. It's a pattern anyone can replicate."
What to Show
  • Slide: architecture diagram (Client → HTTP → MCP Server → Parser)
  • Same protocol Claude Desktop, VS Code, and Copilot Studio all speak
3:00 to 5:00
Segment 3: Live Server Setup
What to Say
  • "One TypeScript file, about 880 lines."
  • "Two dependencies: MCP SDK and Zod. That's it."
  • "Drop a ZIP, run npm run dev, auto-extracts and parses."
  • "Built-in API key auth. Set the key via environment variable, and the server validates every request."
What to Show
  • GitHub repo: single src/index.ts, package.json
  • Terminal: MCP_API_KEY=my-key npm run dev → extracted, tools registered, auth enabled
  • Browser: localhost:3001/health → JSON response (no auth needed)
5:00 to 7:00
Segment 4: Live Copilot Studio Connection
What to Say
  • "Dev tunnel gives us a public HTTPS URL."
  • "In Copilot Studio: Tools → Add tool → MCP → paste URL."
  • "Set authentication to API key, choose Header, enter the header name and key value."
  • "Auto-discovers all 25 tools. Zero config, zero connector code."
  • "Enable generative orchestration so the agent picks the right tool."
What to Show
  • Terminal: devtunnel host → copy URL
  • CS UI: Tools → Add tool → MCP → paste URL + /mcp
  • Auth config: API key → Header → header name → key value
  • Tool discovery list: 25 tools appear automatically
  • Generative orchestration toggle
