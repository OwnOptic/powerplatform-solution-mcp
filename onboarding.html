Get Running in 5 Minutes 0 / 6

1
Install Node.js 18+
Runtime for the MCP server. Check with node --version

Download from nodejs.org. Pick the LTS version. Install and restart your terminal.

Verify:

node --version
npm --version

Both should return version numbers. Node must be 18 or higher.

2
Clone the repository
Get the source code from GitHub
git clone https://github.com/OwnOptic/powerplatform-solution-mcp.git
cd powerplatform-solution-mcp
3
Install dependencies
Just 2 packages: MCP SDK + Zod
npm install

This installs exactly 2 runtime dependencies. Nothing else.

4
Drop your solution ZIP
Export from make.powerapps.com and copy into solutions/

How to export:

  1. Go to make.powerapps.com
  2. Select your Environment (top-right)
  3. Click Solutions in the sidebar
  4. Select your solution, click Export
  5. Choose Unmanaged (recommended for full source)
  6. Download the .zip file

Then drop it in:

# Windows (drag and drop into solutions/ folder)
# Or via terminal:
copy C:\Users\you\Downloads\MySolution.zip .\solutions\

You can drop multiple ZIPs. Each becomes a separate explorable solution.

5
Start the server
Auto-extracts ZIPs, registers 24 tools, starts listening
npm run dev

Expected output:

Power Platform Solution Explorer MCP Server
──────────────────────────────────────────
ZIP drop folder: ./solutions
Extracted to:    ./extracted
✓ Extracted: MySolution.zip → MySolution
MCP endpoint:    http://localhost:3001/mcp
Health check:    http://localhost:3001/health
Tools:           24
Solutions:       MySolution
6
Verify with health check
Confirm the server is running and tools are registered

Open in browser or run:

curl http://localhost:3001/health

Expected response:

{"status":"ok","server":"Power Platform Solution Explorer MCP","version":"2.0.0","tools":24,"solutions":["MySolution"]}

You're ready. Connect any MCP client to http://localhost:3001/mcp

Connect to Copilot Studio 4 steps

1
Start a dev tunnel
Copilot Studio needs a public URL to reach your local server

Install (one-time):

# Windows
winget install Microsoft.devtunnel

# Mac
brew install --cask devtunnel

# Linux
curl -sL https://aka.ms/DevTunnelCliInstall | bash

Login (one-time):

devtunnel user login -g

Start tunnel:

devtunnel host -p 3001 --allow-anonymous

Copy the URL. Your MCP endpoint: https://<tunnel-id>-3001.euw.devtunnels.ms/mcp

2
Register MCP tool in Copilot Studio
Add the MCP server as a tool source
  1. Go to copilotstudio.microsoft.com
  2. Open your agent (or create a new one)
  3. Click Tools in the left sidebar
  4. Click Add a tool then Model Context Protocol
  5. Fill in:
    • Server name: Power Platform Solution Explorer
    • Server URL: your tunnel URL + /mcp
    • Authentication: None (for local dev)
  6. Click Next. Copilot Studio connects and discovers all 24 tools.
  7. Enable all tools, click Add
3
Enable Generative Orchestration
Let the agent automatically pick the right tool per question
  1. Go to your agent's Settings
  2. Click Generative AI
  3. Enable Generative orchestration
  4. Save
4
Test in the chat pane
Ask questions and watch the agent pick tools automatically

Key Design Decisions

Stateless factory pattern: Each HTTP POST creates a fresh McpServer instance. Required for Copilot Studio's Streamable HTTP transport.

Zero external dependencies beyond MCP SDK and Zod. XML parsing uses regex (Power Platform solution XMLs are well-structured and predictable).

Auto-extract on startup: Drop ZIPs in solutions/, start server, done. Invalid ZIPs (no solution.xml) are automatically cleaned up.

Cross-platform: Works on Windows (PowerShell Expand-Archive), Mac, and Linux (unzip).

70+ component type codes mapped from the Dataverse solutioncomponent entity.

Troubleshooting

"npm run dev" fails with "tsx not found"
Run npm install first. tsx is a dev dependency that gets installed automatically.
Server starts but shows "Solutions: none"
You haven't dropped any ZIP files in solutions/ yet, or the ZIPs aren't valid Power Platform solutions. They must contain a solution.xml at the root. Make sure you're exporting from Solutions in make.powerapps.com.
"EADDRINUSE: address already in use :::3001"
Another process is using port 3001. Kill it with npx kill-port 3001, or use a different port: PORT=3002 npm run dev
ZIP extracted but shows "not a Power Platform solution"
The ZIP doesn't contain solution.xml at the root. Make sure you're exporting from Solutions in make.powerapps.com, not downloading something else.
Dev tunnel not working
Check: (1) MCP server is running (curl http://localhost:3001/health), (2) tunnel shows "Ready to accept connections", (3) use the full URL with /mcp at the end, (4) if "Forbidden", make sure you used --allow-anonymous.
Copilot Studio can't discover tools
Check: (1) URL ends with /mcp (not /health), (2) tunnel is working (open https://tunnel-url/health in browser), (3) you selected Model Context Protocol (not REST API or OpenAPI) when adding the tool.
Copilot Studio discovers tools but doesn't use them
Enable Generative orchestration in your agent's settings. Also make sure tools are enabled (checkboxes in the Tools panel). Try being explicit: "Use the list_flows tool to show me all flows."
Can I use this without Copilot Studio?
Yes. Any MCP-compatible client works: Claude Desktop, VS Code with MCP extension, or any custom client. Point it to http://localhost:3001/mcp. No tunnel needed for local clients.
